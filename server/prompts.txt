Please create login service using the db table Users. This should accept email and password. Check whether there is a matching document in Users collection. If yes, it should return a Response object (in response.model.ts) with code 200 and appropriate message. No error message. data should be the matching User document. If matching document could not be located in Users collection, create Response object with code 404. Use appropriate error message and message. Then return this Response object. Please create agentRegistration service. An agent is a user (Users table) with additional information like agency name etc.
Agent collection has following information. 1) agencyName 2) website 3) experience - number of years of experience - integer 4) licenceNumber 5) panNumber 6) specialities - text of size 500 7) UserId - reference to Users table. In addition to the first 6 details, email will also be sent to the service. Service will check whether there is a user with given email. If such a user is found, service will check whether there is already an agent with that user id. If such an agent is found, service returns Response object with code 409 (means agent already exists). If such an agent could not be found, it will add an agent document into agents collection with above 6 details + UserId. After that the type field of corresponding user document will be updated to "agent". Then, the service returns Response object with code 200 & data as agent document.
